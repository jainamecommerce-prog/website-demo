Building a Website with Claude Code

Step 1

CLAUDE.md — Your Project Instructions File

The first thing Claude Code reads when it opens your project. It sets the rules, context, and behavior for the entire session.

What is CLAUDE.md?
A markdown file at the root of your project. Claude reads it automatically every session — brand details, rules, and project context all in one place.
Step 2

Invoking the Frontend-Design Skill

Skills are pre-built workflows that live inside Claude Code. They give Claude a specialized set of instructions for a specific type of task — like designing production-grade UIs.

What is a Skill?
A markdown file at ~/.claude/skills/skill-name/skill.md. The frontend-design skill gives Claude a strict design philosophy — no generic output, production-ready code only.
Skill location on your machine
C:\Users\ADMIN\.claude\skills\frontend-design\skill.md
Invoke the skill — type this in Claude Code chat
/frontend-design
Step 3

Borrow Design Elements, Not Entire Sites

You're not copying these websites — you're picking one specific thing you like from each. A button style, a glow effect, a card layout. Spot it, describe it to Claude, and it will recreate that element in the uniThink brand.

How this works:
Browse any of the sites below, find one design detail that catches your eye — a hover effect, a gradient text heading, a glassmorphism card — then tell Claude: "I want something like [that specific thing] from [site]." Claude adapts it to your colors, fonts, and style. You get the look, not a copy.

Step 4

Individual Components

Build your website component by component. Each prompt to Claude targets one specific part of the page — a navbar, a hero, a features section. This keeps the output focused and the result easy to review and adjust.

What is a component?
A component is a self-contained block of your webpage — the navbar, the hero banner, a testimonials row, the footer. By asking Claude to build one at a time, you stay in full control of the layout and can refine each piece before moving on.
Common website components
Navbar — Logo on left, nav links on right, sticky on scroll Hero — Bold headline, subtext, CTA button, background glow Features — 3-column grid, icon + heading + one-line description Testimonials — Cards with quote, name, and role Pricing — 2–3 tier cards, highlight the recommended plan Footer — Links, socials, copyright line
How to prompt Claude for each component
"Add a navbar. Logo text 'uniThink' on the left using the brand gradient. Nav links on the right: Home, About, Pricing, Contact. Sticky on scroll, dark background." "Now add a hero section below the navbar. Headline: 'Think Bigger.' Muted subtext underneath. A gradient CTA button. Subtle glow effect behind the headline."
The approach: One component per prompt. Open the result in the browser after each one, decide what needs to change or what comes next, then give Claude the next instruction. Small steps, full control.
